## 📚 3. 배워야 할 것들

> 단기간 암기 ❌ — 스터디 과정을 통해 자연스럽게 익힐 것

### Python 기초 문법
코드를 직접 짜는 것보다 **읽고 흐름을 파악하는 수준**이 목표입니다.

| 구분 | 내용 | 수준 |
|------|------|------|
| 필수 | 변수 / 조건문 / 반복문 / 함수(def) | 코드 흐름 읽기 |
| 필수 | 클래스 / self / `__init__` | AI 코드 전체가 클래스 구조 |
| 필수 | 리스트 / 딕셔너리 | 데이터 담는 기본 구조 |
| 권장 | 리스트 컴프리헨션 | AI 코드에 자주 등장 |

### 라이브러리
메서드를 외우는 것이 아니라 **패턴을 익히는 것**이 목표입니다.

| 라이브러리 | 용도 | 익혀야 할 수준 |
|-----------|------|--------------|
| NumPy | 배열(텐서) 다루기 | shape / reshape / 슬라이싱 패턴 |
| PyTorch | AI 모델 구현 | forward / loss / optimizer 4줄 패턴 |
| Pandas | 데이터 전처리 | CSV 읽고 정제하는 기본 사용법 |
| HuggingFace Transformers | 모델 불러와 실행 | pipeline으로 5줄 안에 실행하기 |
| LangChain | RAG 파이프라인 | 튜토리얼 따라 RAG 챗봇 만들기 |
| Gradio | 모델에 UI 입혀서 데모 배포 | Interface 함수로 입출력 연결 |

### 도구 (Tools)
코딩 실력보다 이 도구들을 다룰 줄 아는 것이 **취업에서 실질적인 차별점**이 됩니다.

| 도구 | 용도 | 익혀야 할 기능 |
|------|------|--------------|
| GitHub | 코드 버전 관리 + 포트폴리오 | clone / add / commit / push / README 작성 |
| Google Colab | 무료 GPU 실습 환경 | 노트북 생성 / 실행 / 링크 공유 |
| HuggingFace Spaces | 모델 데모 배포 | Gradio 앱 업로드 + 링크 공유 |
| Gradio | AI 모델에 UI 입히기 | Interface 함수로 입출력 연결하기 |

## 📖 7. 논문 읽는 방법

처음부터 끝까지 정독하지 않습니다. 아래 순서를 팀 공식 방법으로 통일합니다.

1. Claude → 프로젝트 → 논문분석 → 해당 주차 논문 채팅창 (ex. 알렉스넷)
2. **3회독 전략**

| 회독 | 읽는 순서 | 목표 |
|------|----------|------|
| 1회독 | 그냥 한번 쭉 "정독"하기 | 문제 / 해결 / 결과 3줄로 요약 |
| 2회독 | 핵심 아이디어 및 전체 구조 중심으로 읽기 | 전체 구조에 대해 설명할 수 있도록 |
| 3회독 | 논문과 코드의 연결성 + 데이터 shape의 변화 흐름 → 직접 코드 구현 후 HuggingFace 업로드 | ① 구조를 어떻게 코드로 옮겼는지 ② 해당 구조의 input/output 데이터가 뭔지 알 수 있도록 |

### 모를 때 해결하는 방법
- 모르는 수식은 일단 넘기고 개념만 이해 — 3회독 때 다시 보면 훨씬 쉬워짐
- 이해 안 되는 섹션은 Claude / ChatGPT에 붙여넣고 '쉽게 설명해줘' 활용
- 논문은 [Papers With Code](https://paperswithcode.com)에서 찾고, 코드는 같이 올라온 GitHub 링크 활용

## 📘 출력 구조 (반드시 이 순서)

### 1. Abstract (3줄 요약)
문제: (한 줄)
방법: (한 줄)
결과: (한 줄)

### 2. Introduction + Conclusion 요약
- 이 논문이 왜 나왔는지
- 기존 방식의 문제점
- 이 논문이 어떤 개선을 했는지

### 3. 핵심 구조 설명 (가장 중요)
논문에서 사용하는 주요 구성 요소들을 설명하라.
(해당 논문에서 새롭게 등장하는 핵심 개념은 (핵심)이라고 표시할 것)

각 요소마다 반드시 아래 4가지를 포함:
1. 이게 뭔지 (비유 먼저)
2. 입력값 (데이터 형태)
3. 출력값 (데이터 형태)
4. 하이퍼파라미터 (사람이 설정하는 값)

### 4. 구조 시각화
논문에 나오는 전체 모델 구조(데이터 흐름)를 다이어그램으로 시각화.
각 단계마다 데이터 형태 변화 포함.

### 5. 논문 ↔ 코드 연결 (PyTorch)
논문에서 설명한 각 구조가 실제 코드에서 어느 줄에 해당하는지 연결.
예) 논문: "ReLU를 Conv layer 뒤에 적용했다" → 코드: nn.ReLU(inplace=True)
전체 모델을 코드로 구현한 예시도 포함.

### 6. 데이터 흐름 추적 (Shape 변화)
형식: (batch, C, H, W) → [Layer] → (변경된 shape)
각 단계마다 shape 변화 + 이유 한 줄 설명
