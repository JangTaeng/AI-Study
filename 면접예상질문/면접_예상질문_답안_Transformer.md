## Q1. Self-Attention과 RNN의 차이를 설명해보세요. 트랜스포머는 왜 RNN보다 빠른가요?

**핵심 차이: "순차 처리" vs "병렬 처리"**

RNN은 문장을 한 단어씩 **순서대로** 처리합니다. "I am a student"라는 문장을 처리할 때, "am"을 계산하려면 "I"가 끝나야 하고, "a"를 계산하려면 "am"이 끝나야 합니다. 즉, 앞 단어의 계산이 끝날 때까지 다음 단어는 기다려야 합니다.

반면 Self-Attention은 모든 단어를 **동시에** 처리합니다. 모든 단어가 서로를 한 번에 바라보고 관계를 계산합니다. GPU의 병렬 연산 능력을 100% 활용할 수 있습니다.

**비유: 전화 vs 회의실**

- **RNN**: 100명이 한 줄로 서서 귓속말로 정보를 전달 — 100번째 사람까지 도달하려면 99단계가 필요
- **Self-Attention**: 100명이 회의실에 모여 서로 동시에 대화 — 한 단계 만에 모두가 모두를 알게 됨

**논문 Table 1 비교 (핵심 지표 2가지)**

| 비교 항목 | Self-Attention | RNN (Recurrent) |
|-----------|---------------|-----------------|
| Sequential Operations (순차 연산 횟수) | **O(1)** | O(n) |
| Maximum Path Length (두 단어 간 경로 길이) | **O(1)** | O(n) |
| Complexity per Layer | O(n² · d) | O(n · d²) |
| 병렬화 가능 여부 | ✅ 완전 병렬 | ❌ 불가능 |

**Maximum Path Length가 왜 중요한가요?**

문장 첫 단어와 마지막 단어가 서로 영향을 주려면 정보가 그 사이 경로를 거쳐야 합니다. 경로가 길수록 **장거리 의존성(long-range dependency)** 학습이 어려워집니다.

- RNN: 첫 단어와 100번째 단어가 연결되려면 100단계를 거쳐야 함 → 신호가 약해짐 (기울기 소실)
- Self-Attention: 첫 단어와 100번째 단어가 **단 1단계**로 직접 연결됨 → 신호 손실 없음

**결론**

Self-Attention은 (1) 모든 단어를 병렬로 처리해 학습 속도가 빠르고, (2) 두 단어 사이 거리가 항상 1이라 장거리 의존성을 잘 학습합니다. RNN의 두 가지 약점을 동시에 해결한 것이 트랜스포머의 본질입니다.

## Q2. Scaled Dot-Product Attention에서 왜 √dk로 나누나요? 안 나누면 어떻게 되나요?

**한 줄 요약: Softmax가 너무 뾰족해지는 것을 방지하기 위함**

**문제 상황: 내적값의 분산 폭발**

Attention의 핵심 연산은 Query와 Key의 내적(dot product)입니다. Q와 K가 평균 0, 분산 1인 독립 확률 변수일 때:

```
Q · K = q₁k₁ + q₂k₂ + ... + q_{dk}k_{dk}    ← dk개의 항을 더함
```

확률 통계적으로, **dk개의 독립 변수를 더하면 분산이 dk배 증가**합니다.

| dk | 내적값의 표준편차 (대략) |
|----|---------------------|
| 8  | √8 ≈ 2.83 |
| 64 | √64 = 8 |
| **512** | **√512 ≈ 22.6** ← 매우 큰 값들 발생 |

**그래서 무슨 일이 일어나는가? — Softmax의 함정**

Softmax는 입력값의 차이를 **지수적으로** 증폭합니다. 입력값이 크면 가장 큰 값에 확률이 거의 100% 몰리고, 나머지는 0에 가까워집니다.

```
입력 [1, 2, 3]      → softmax → [0.09, 0.24, 0.67]   (부드러움)
입력 [10, 20, 30]   → softmax → [0.00, 0.00, 1.00]   (극단적으로 뾰족!)
```

뾰족한 softmax = **gradient가 거의 0** → 학습이 멈춤 (기울기 소실)

**해결책: √dk로 나누어 분산을 다시 1로 맞추기**

```
원래 내적의 분산: dk
√dk로 나누면 분산: dk / (√dk)² = dk / dk = 1  ✅
```

이렇게 하면 dk가 아무리 커져도 내적값의 크기가 일정하게 유지되어, softmax가 적당히 부드러운 분포를 출력합니다.

**비유: 볼륨 조절기**

dk가 커진다는 건 마이크가 점점 민감해지는 것과 같습니다. 그대로 두면 볼륨이 폭발하니까 √dk로 나눠서 볼륨을 정상 수준으로 맞추는 것입니다.

**한 줄 요약**

> dk가 커지면 내적값의 분산이 dk배 증가 → softmax가 한 곳에 몰림 → gradient 소실 → 학습 실패. 이를 방지하기 위해 분산을 1로 정규화하는 것이 √dk 스케일링입니다.

## Q3. Multi-Head Attention은 왜 한 개의 큰 어텐션을 쓰지 않고 여러 개로 쪼개나요?

**핵심 아이디어: 서로 다른 "관점"으로 문장을 동시에 보기**

한 문장 안에는 여러 종류의 관계가 동시에 존재합니다.
- 주어-동사 관계 (문법적)
- 대명사가 가리키는 대상 (지시 관계)
- 의미적 유사성 (semantic)
- 위치적 가까움 (positional)

만약 어텐션이 **하나뿐**이라면, 이 모든 관계를 한 번에 평균 내서 학습해야 합니다. 결과적으로 **모든 정보가 뭉개져서** 어떤 관계도 제대로 포착하지 못합니다.

**비유: 안경 8개**

- 단일 헤드 = 안경 1개 (한 가지 시야로만 봄)
- 멀티 헤드 = 안경 8개 (각각 다른 필터를 가지고 동시에 봄)
  - 1번 안경: 문법 구조에 집중
  - 2번 안경: 의미적 유사성에 집중
  - 3번 안경: 멀리 떨어진 단어 관계에 집중
  - ... (헤드마다 다른 패턴 학습)

마지막에 8개의 시야를 합쳐서(concat) 통합된 이해를 만듭니다.

**논문에서 쓴 표현: "Representation Subspace"**

논문은 이렇게 말합니다: 각 헤드는 서로 다른 **표현 부분공간(representation subspace)** 에서 정보를 학습한다고요. 즉, 같은 단어라도 헤드마다 다른 "측면"을 보고 있다는 뜻입니다.

**단일 헤드 vs 멀티 헤드 비교**

| 비교 항목 | 단일 헤드 (dmodel=512) | 멀티 헤드 (8 × 64) |
|-----------|---------------------|------------------|
| Q, K, V 차원 | 512 | 헤드당 64 |
| 학습 가능한 패턴 | 1가지 (평균) | 8가지 (다양함) |
| 총 파라미터 수 | 비슷함 | 비슷함 (차원을 쪼갰을 뿐) |
| 표현력 | 제한적 | **풍부함** |
| 연산량 | 비슷함 | 비슷함 (병렬화로 추가 비용 거의 없음) |

**중요한 트릭: 차원을 늘리는 게 아니라 "쪼개는" 것**

- dmodel = 512
- 헤드 수 h = 8
- 헤드당 차원 dk = dv = 512 / 8 = **64**

전체 파라미터 수는 단일 헤드와 거의 같습니다. **공짜로 표현력만 늘리는** 영리한 설계입니다.

**논문 Figure 3, 4, 5에서 본 시각화 결과**

논문의 어텐션 시각화를 보면 헤드마다 정말로 다른 패턴을 학습합니다.
- 어떤 헤드는 "it"이 가리키는 명사를 정확히 찾음 (지시 관계)
- 어떤 헤드는 동사와 그 목적어를 연결함 (문법 관계)
- 어떤 헤드는 같은 어근의 단어들을 묶음 (형태론적 관계)

**한 줄 요약**

> 단일 헤드는 모든 관계를 평균 내어 정보가 뭉개지지만, 멀티 헤드는 서로 다른 부분공간에서 각자 다른 패턴(문법, 의미, 지시 관계 등)을 동시에 학습해 표현력이 훨씬 풍부합니다.

## Q4. 디코더의 Self-Attention에서는 왜 마스킹(Masking)을 사용하나요? 어떻게 구현되나요?

**핵심 이유: Auto-regressive(자기회귀) 속성을 지키기 위함**

**Auto-regressive란?**

번역 모델이 "I am a student"를 "나는 학생입니다"로 번역할 때, 디코더는 한 단어씩 차례로 생성합니다.

```
1단계: <start>           →  "나는"
2단계: <start> 나는        →  "학생"
3단계: <start> 나는 학생    →  "입니다"
```

**핵심: i번째 단어를 예측할 때는 1 ~ (i-1)번째 단어만 봐야 합니다.** i번째 이후 단어는 아직 만들어지지 않았기 때문이죠.

**문제: 학습 시점에는 정답이 통째로 들어감**

학습할 때는 효율을 위해 정답 문장 ("나는 학생입니다")을 한꺼번에 디코더에 입력합니다. 그런데 Self-Attention은 모든 단어가 서로를 보는 구조라서, 그냥 두면 "나는"을 예측할 때 디코더가 **이미 정답인 "학생", "입니다"를 컨닝**할 수 있습니다.

이걸 **information leakage(정보 누출)** 이라고 합니다. 학습 때는 시험 답안을 미리 본 셈이라 잘 동작하는 것처럼 보이지만, 실제 추론 때는 미래 단어가 없으니 성능이 폭망합니다.

**해결책: Look-Ahead Mask (미래 가리기)**

미래 단어의 attention score를 **−∞ (실제로는 매우 큰 음수)** 로 설정합니다. 그러면 softmax를 통과한 후 그 위치의 가중치가 **0**이 되어 미래 단어가 결과에 영향을 주지 못합니다.

```python
# 마스크 행렬 (4단어 예시) — 1은 막기, 0은 통과
mask = [
    [0, 1, 1, 1],   # 1번 단어: 자기만 봄
    [0, 0, 1, 1],   # 2번 단어: 1, 2 봄
    [0, 0, 0, 1],   # 3번 단어: 1, 2, 3 봄
    [0, 0, 0, 0],   # 4번 단어: 1, 2, 3, 4 모두 봄
]

# Attention score에 적용
scores = scores.masked_fill(mask == 1, float('-inf'))
attn = softmax(scores)  # -inf 자리는 0이 됨
```

**시각화: 삼각형 모양의 마스크**

```
         단어1  단어2  단어3  단어4
단어1 →   ✅    ❌    ❌    ❌
단어2 →   ✅    ✅    ❌    ❌
단어3 →   ✅    ✅    ✅    ❌
단어4 →   ✅    ✅    ✅    ✅
```

이걸 **하삼각 마스크(lower triangular mask)** 라고 부릅니다.

**비유: 시험장에서 옆 사람 답안 가리기**

학생이 시험을 볼 때, 자기 답을 쓰기 전에 뒤에 있는 친구의 답안을 보면 안 됩니다. Look-Ahead Mask는 시험관이 가림판으로 미래 답안을 가려주는 역할입니다.

**왜 인코더 Self-Attention에는 마스크가 없나?**

인코더는 입력 문장 전체를 이해하는 게 목적이라 모든 단어가 서로를 봐도 됩니다. 마스크는 오직 디코더의 첫 번째 Self-Attention에서만 사용됩니다 (그래서 논문에서는 **Masked Multi-Head Attention** 이라고 부릅니다).

**한 줄 요약**

> 디코더는 단어를 하나씩 순차 생성하는 auto-regressive 모델이므로, 학습 시 미래 단어를 보는 컨닝을 막기 위해 attention score에 −∞를 넣어 softmax 후 가중치를 0으로 만드는 마스킹을 사용합니다.

## Q5. Positional Encoding이 왜 필요한가요? 학습형(learned) 대신 sin/cos 함수를 쓴 이유는?

**왜 필요한가? — 트랜스포머는 "순서 감각"이 없다**

RNN은 단어를 한 개씩 순서대로 처리하므로 자연스럽게 단어 위치 정보를 갖습니다. 하지만 Self-Attention은 모든 단어를 **동시에 평등하게** 바라봅니다. 이것의 부작용이 바로 **순서 정보가 사라진다**는 것입니다.

**충격적인 사실: 단어 순서를 바꿔도 결과가 같다!**

Self-Attention만 사용하면 다음 두 문장의 출력이 **완전히 동일**합니다.

- "나는 너를 사랑한다"
- "너를 나는 사랑한다"
- "사랑한다 나는 너를"

자연어 처리에서 이건 치명적인 문제입니다. "개가 사람을 물었다" vs "사람이 개를 물었다"가 같다고 처리되면 안 되니까요.

**해결책: 위치 정보를 직접 임베딩에 더해주기**

각 위치 (1번, 2번, 3번 ...)에 대해 고유한 벡터를 만들고, 단어 임베딩에 **더해서** 입력합니다.

```
입력 = WordEmbedding("나는") + PositionalEncoding(위치=1)
입력 = WordEmbedding("너를") + PositionalEncoding(위치=2)
```

이렇게 하면 같은 단어라도 위치마다 다른 입력값이 되어 모델이 순서를 구별할 수 있습니다.

**Sin/Cos 공식**

```
PE(pos, 2i)   = sin(pos / 10000^(2i/dmodel))
PE(pos, 2i+1) = cos(pos / 10000^(2i/dmodel))
```

- pos: 단어 위치 (0, 1, 2, ...)
- i: 임베딩 차원 인덱스
- 짝수 차원에는 sin, 홀수 차원에는 cos 적용

**Sin/Cos vs 학습형(Learned) Positional Embedding 비교**

| 비교 항목 | Sin/Cos (고정) | Learned (학습형) |
|-----------|---------------|-----------------|
| 학습 파라미터 | ❌ 없음 | ✅ 있음 |
| 학습 시 본 길이를 넘는 시퀀스 처리 | ✅ **가능** | ❌ 불가능 |
| 두 위치의 상대적 거리 표현 | ✅ 자연스러움 (선형 변환으로 표현 가능) | ❌ 불가능 |
| 성능 | 거의 동일 | 거의 동일 |

**왜 sin/cos이 "학습 시 보지 못한 길이"도 처리 가능한가?**

학습형 임베딩은 위치별로 룩업 테이블을 만들어 학습합니다. 학습 때 50개까지만 봤다면 51번째 위치의 벡터는 존재하지 않습니다.

반면 sin/cos는 **수학 함수**라서 어떤 위치 값이 들어와도 계산만 하면 됩니다. 5000번째 위치든 10000번째 위치든 공식에 넣으면 즉시 벡터가 나옵니다.

**중요한 추가 장점: 상대적 위치 관계의 자연스러운 표현**

삼각함수의 성질로 인해, **두 위치의 거리(k)가 일정하면 PE(pos+k)는 PE(pos)의 선형 변환으로 표현 가능**합니다. 즉, "5칸 떨어진 관계"가 모든 위치에서 일관되게 표현됩니다. 모델이 "상대적 거리"를 쉽게 학습할 수 있게 해주는 강력한 특성입니다.

**비유: 시계의 분침과 시침**

sin/cos 함수는 주기적으로 회전합니다. 시계의 분침과 시침처럼, 차원마다 회전 속도가 다른 여러 개의 바늘을 조합하면 **어떤 시각이든 고유하게 표현**됩니다. PE는 이런 "여러 주기의 시계"를 합쳐 위치를 인코딩하는 것입니다.

**논문의 실험 결과**

논문 Table 3에서 sin/cos과 learned 방식을 비교했는데 **성능 차이는 거의 없었습니다**. 그래서 굳이 학습 파라미터를 추가하지 않고, 더 일반적인(긴 시퀀스에 강건한) sin/cos 방식을 채택했습니다.
