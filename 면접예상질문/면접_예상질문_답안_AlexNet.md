## Q1. AlexNet이 이전 모델들보다 혁신적이었던 이유 3가지를 설명하세요.

**① ReLU 활성화 함수 도입**
기존 모델들은 `tanh`나 `sigmoid`를 사용했는데, 이 함수들은 값이 커질수록 기울기가 0에 가까워지는 **기울기 소실(Vanishing Gradient)** 문제가 있었습니다. AlexNet은 `f(x) = max(0, x)`인 ReLU를 도입해 이 문제를 해결하고, `tanh` 대비 **6배 빠른 학습 속도**를 달성했습니다.

**② Dropout으로 과적합 방지**
학습 중 뉴런을 50% 확률로 무작위로 꺼버리는 Dropout을 처음으로 대규모 CNN에 적용했습니다. 덕분에 모델이 특정 패턴에 과도하게 의존하지 않게 되어 **일반화 성능**이 크게 향상되었습니다.

**③ GPU 2개를 이용한 대규모 병렬 학습**
당시 단일 GPU로는 메모리가 부족해 대규모 CNN 학습이 불가능했습니다. AlexNet은 GPU 2개에 모델을 나눠 올리는 **병렬 학습 구조**를 설계해, 120만 장의 이미지로 깊은 신경망을 현실적인 시간 안에 학습시킬 수 있었습니다.

## Q2. ReLU를 사용했을 때 tanh 대비 학습이 왜 빨라지나요?

**핵심 원인: 기울기 소실(Vanishing Gradient) 문제**

`tanh` 함수는 입력값이 매우 크거나 작아지면 기울기(미분값)가 **거의 0**에 수렴합니다.

```
tanh 미분: 1 - tanh²(x)  →  x가 크면 ≈ 0
```

역전파(Backpropagation)는 기울기를 곱하면서 앞 레이어로 전달하는데, 기울기가 0에 가까우면 **앞쪽 레이어까지 신호가 거의 전달되지 않습니다.** 레이어가 깊을수록 이 문제는 심각해집니다.

반면 ReLU는 양수 구간에서 기울기가 항상 **1**로 고정됩니다.

```
ReLU 미분: x > 0이면 1, x ≤ 0이면 0
```

기울기가 죽지 않으니 역전파 신호가 깊은 레이어까지 잘 전달되고, 그 결과 학습이 훨씬 빠르게 수렴합니다.

| 비교 항목 | tanh | ReLU |
|-----------|------|------|
| 기울기 범위 | 0 ~ 1 (포화 시 ≈ 0) | 0 또는 1 |
| 기울기 소실 | 발생함 | 발생 안 함 (양수 구간) |
| 학습 속도 | 느림 | **6배 빠름** |
| 연산 복잡도 | 지수함수 계산 필요 | `max(0, x)`로 단순 |

## Q3. Dropout은 어떻게 과적합을 방지하고, 테스트 시에는 어떻게 동작하나요?

**학습 시 동작**

매 학습 스텝마다 각 뉴런을 확률 `p=0.5`로 무작위로 **꺼버립니다(출력을 0으로 만듦).** 그 결과:
- 모델이 특정 뉴런 조합에 의존하지 못하게 됩니다.
- 매번 다른 부분 네트워크(sub-network)가 학습되는 효과가 생깁니다.
- 마치 수많은 모델을 앙상블(Ensemble)하는 것과 유사한 효과를 냅니다.

```
학습 시: 출력 = 원래값 × Bernoulli(p=0.5)  →  50% 뉴런이 0
```

**테스트(추론) 시 동작**

테스트 시에는 모든 뉴런을 다 사용합니다. 하지만 학습 때는 절반의 뉴런만 켜져 있었으므로, 출력 스케일을 맞추기 위해 **출력값에 0.5를 곱합니다.**

```
테스트 시: 출력 = 원래값 × 0.5  →  학습 시 기댓값과 동일하게 맞춤
```

> 💡 현대 PyTorch에서는 이를 자동으로 처리합니다.
> 학습 시 출력을 `1/p`로 나눠서 스케일을 맞추는 **Inverted Dropout** 방식을 사용하기 때문에, 테스트 시 별도의 0.5 곱셈이 필요 없습니다.

## Q4. AlexNet에서 GPU를 2개 사용한 이유와, GPU 간에 언제 통신이 일어나는지 설명하세요.

**GPU를 2개 사용한 이유: 메모리 한계**

당시 사용한 GPU(GTX 580)는 VRAM이 **3GB**밖에 없었습니다. AlexNet의 파라미터 수는 약 6000만 개로, 단일 GPU에 올리기에는 메모리가 부족했습니다. 그래서 모델을 **두 GPU에 절반씩 나눠서** 올렸습니다.

```
GPU 0: 필터 절반 (예: 48개)
GPU 1: 필터 절반 (예: 48개)
         → 합쳐서 96개 필터
```

**GPU 간 통신이 일어나는 시점**

대부분의 레이어에서는 각 GPU가 자기 절반만 처리하고 서로 통신하지 않습니다. 통신이 일어나는 레이어는 단 하나입니다.

| 레이어 | GPU 간 통신 여부 |
|--------|----------------|
| Conv1, Conv2 | ❌ 없음 (각자 처리) |
| **Conv3** | ✅ **있음** — 양쪽 GPU의 특징 맵을 모두 받아서 처리 |
| Conv4, Conv5 | ❌ 없음 (각자 처리) |
| FC1, FC2, FC3 | ❌ 없음 (각자 처리) |

Conv3에서만 GPU 0과 GPU 1이 서로의 데이터를 교환합니다. 이를 통해 두 GPU의 특징이 합쳐지며, 더 풍부한 표현을 학습할 수 있게 됩니다.

## Q5. Max Pooling과 Overlapping Pooling의 차이는 무엇이고, AlexNet은 왜 Overlapping Pooling을 선택했나요?

**일반 Max Pooling**

풀링 창(kernel)이 서로 **겹치지 않게** 이동합니다. 이동 간격(stride)이 창 크기와 같습니다.

```
일반 풀링: stride(s) = kernel size(z)
예: z=2, s=2  →  창이 딱 붙어서 이동 (겹침 없음)
```

**Overlapping Pooling (AlexNet)**

풀링 창이 서로 **겹치도록** 이동합니다. 이동 간격(stride)이 창 크기보다 작습니다.

```
Overlapping 풀링: stride(s) < kernel size(z)
AlexNet: z=3, s=2  →  창이 1픽셀씩 겹치며 이동
```

**왜 Overlapping Pooling을 선택했나요?**

| 비교 항목 | 일반 Max Pooling | Overlapping Pooling |
|-----------|-----------------|-------------------|
| 겹침 여부 | 없음 (`s = z`) | 있음 (`s < z`) |
| 정보 손실 | 상대적으로 많음 | 적음 |
| 과적합 | 상대적으로 높음 | **낮음** |
| AlexNet 결과 | — | top-1 오류 **0.4%↓**, top-5 오류 **0.3%↓** 개선 |

창이 겹치면서 이동하면 인접한 위치의 정보가 더 많이 보존됩니다. 논문에서도 Overlapping Pooling이 과적합을 줄이는 데 효과적이었다고 명시했으며, 실험을 통해 일반 풀링보다 성능이 향상됨을 확인했습니다.
