## Q1. ResNet이 해결하려 한 핵심 문제가 무엇인가요?

**핵심 문제: 성능 저하(Degradation) 문제**

ResNet이 등장하기 전, 딥러닝에서는 "층을 더 쌓을수록 성능이 좋아진다"는 것이 상식이었습니다. 그런데 실험해보니 56층 네트워크가 20층 네트워크보다 **학습 오류(training error)가 오히려 더 높은** 현상이 발견됐습니다. 이것이 바로 성능 저하(Degradation) 문제입니다.

**기울기 소실(Vanishing Gradient) 문제와의 차이**

| 비교 항목 | 기울기 소실 | 성능 저하(Degradation) |
|-----------|------------|----------------------|
| 원인 | 역전파 중 기울기가 0에 수렴 | 최적화 자체가 어려워짐 |
| 증상 | 학습이 아예 시작 안 됨 | 학습은 되지만 깊을수록 성능이 나빠짐 |
| 해결책 | BN, ReLU, 가중치 초기화 | **Residual Learning (ResNet)** |
| 과적합 여부 | 무관 | ❌ 과적합이 아님 — 훈련 오류 자체가 높음 |

당시 배치 정규화(Batch Normalization)로 기울기 소실은 이미 어느 정도 해결된 상태였습니다. 즉, 성능 저하는 기울기 소실과는 **별개의 문제**였고, ResNet은 이 문제를 타깃으로 설계되었습니다.

**왜 더 깊은 네트워크가 얕은 네트워크보다 나쁠 수 있나요?**

이론적으로, 깊은 네트워크는 추가된 레이어가 **항등 함수(identity mapping)** 만 학습하면 얕은 네트워크와 동등해질 수 있습니다. 하지만 실험에서는 현재의 최적화 방법(SGD 등)이 이를 제대로 학습하지 못한다는 것이 밝혀졌습니다. 레이어가 항등 함수를 학습하는 것조차 어렵다는 것이죠. ResNet은 이 점을 구조적으로 해결했습니다.

## Q2. Residual(잔차) 학습의 핵심 아이디어를 수식 없이 설명해보세요.

**비유: 시험 답안 수정**

기존 방식은 레이어에게 "백지 상태에서 정답을 처음부터 써라"라고 요구하는 것과 같습니다. 반면 Residual 학습은 "이미 있는 답안에서 **틀린 부분만 찾아서 수정**해라"라고 요구하는 것입니다.

```
기존 방식: H(x) 를 처음부터 학습  →  어려움
ResNet 방식: F(x) = H(x) - x 만 학습  →  상대적으로 쉬움
             최종 출력 = F(x) + x  (수정분 + 원본)
```

**왜 이게 더 쉬울까요?**

만약 현재 레이어가 이미 충분히 잘 동작하고 있다면, 추가 레이어가 배워야 할 최적의 답은 "아무것도 바꾸지 마라", 즉 **수정분 F(x) = 0** 입니다. 0을 학습하는 것은 복잡한 함수를 처음부터 학습하는 것보다 훨씬 쉽습니다.

**핵심 이점**

- 레이어를 추가해도 최악의 경우 F(x)=0이 되어 성능이 유지됨 → Degradation 문제 해결
- "변화가 필요한 부분"만 집중적으로 학습하므로 최적화가 쉬워짐
- 실험에서도 학습된 잔차 함수의 응답값이 대체로 작았음 → 대부분의 레이어가 미세 수정만 하고 있다는 증거

## Q3. Shortcut connection이 역전파(backpropagation)에서 어떤 이점을 가져오나요?

**역전파 기본 원리**

역전파는 출력 쪽 손실(loss)에서 시작해 기울기를 **레이어를 거슬러 올라가며 곱해서** 전달합니다. 레이어가 많을수록 기울기가 점점 작아져 앞쪽 레이어까지 신호가 전달되지 않는 기울기 소실이 발생합니다.

**Shortcut Connection이 만드는 추가 경로**

숏컷 연결이 있으면 기울기가 흐를 수 있는 경로가 2개가 됩니다.

```
일반 경로: 손실 → Conv → BN → ReLU → Conv → BN → (앞 레이어)
숏컷 경로: 손실 ─────────────────────────────→ (앞 레이어)  ← 바로 전달!
```

숏컷 경로는 레이어를 거치지 않으므로 기울기가 **거의 손실 없이** 그대로 앞쪽 레이어에 전달됩니다.

**결과**

| 비교 항목 | 일반 네트워크 | ResNet |
|-----------|-------------|--------|
| 기울기 전달 경로 | 1개 (레이어 통과만) | 2개 (레이어 + 숏컷) |
| 깊은 층의 기울기 | 지수적으로 소멸 가능 | 숏컷으로 직접 전달 |
| 학습 안정성 | 층이 깊을수록 불안정 | 매우 안정적 |

덕분에 ResNet은 152층이라는 극단적인 깊이에서도 안정적으로 학습할 수 있었습니다.

## Q4. Bottleneck Block에서 1×1 conv를 사용하는 이유가 무엇인가요?

**1×1 conv의 역할: 채널 수 조절**

1×1 conv는 이미지의 공간적 크기(H, W)는 그대로 두고 **채널 수만** 줄이거나 늘릴 수 있습니다. 이를 활용해 Bottleneck Block은 3×3 conv 앞뒤로 채널을 압축·복원합니다.

```
입력 (256채널)
    ↓ 1×1 conv  →  채널 축소 (256 → 64)
    ↓ 3×3 conv  →  핵심 특징 추출 (64채널 유지)
    ↓ 1×1 conv  →  채널 복원 (64 → 256)
출력 (256채널)
```

**계산량 비교: 왜 효율적인가?**

3×3 conv의 연산량은 입력 채널 수에 비례합니다. 채널을 줄인 상태에서 3×3 conv를 수행하면 계산량이 크게 줄어듭니다.

| 방식 | 구성 | 연산량 (대략) |
|------|------|-------------|
| Basic Block | 3×3 conv(256ch) × 2 | 256 × 256 × 3 × 3 × 2 |
| Bottleneck Block | 1×1 + 3×3 + 1×1 | **훨씬 적음** — 3×3은 64ch에서만 수행 |

**결론**

1×1 conv로 채널을 먼저 줄여(병목, Bottleneck) 계산 비용을 낮추고, 3×3 conv로 핵심 특징을 추출한 뒤, 다시 1×1 conv로 채널을 복원합니다. 덕분에 ResNet-50은 ResNet-34보다 층이 더 많지만 계산량은 비슷하게 유지할 수 있습니다.

## Q5. ResNet-50과 ResNet-34의 구조적 차이점은 무엇이고, 왜 다른 블록 설계를 사용하나요?

**블록 구조 비교**

| 항목 | ResNet-34 (Basic Block) | ResNet-50 (Bottleneck Block) |
|------|------------------------|------------------------------|
| 구성 | 3×3 conv → 3×3 conv | 1×1 conv → 3×3 conv → 1×1 conv |
| 레이어 수 | 2개 | 3개 |
| 채널 변화 | 입출력 동일 | 입력→축소→복원 (예: 256→64→256) |
| 파라미터 수 | 적음 | **많음** (더 넓은 표현력) |
| 연산량 | 비교적 많음 | **효율적** (1×1 병목으로 절감) |

**왜 ResNet-50은 Bottleneck을 사용하나요?**

층이 늘어날수록 처리해야 할 채널 수도 커집니다(256, 512, 1024, 2048). 이 상태에서 Basic Block을 그대로 쌓으면 3×3 conv를 넓은 채널에 반복 적용하게 되어 **연산량이 폭발적으로 증가**합니다.

Bottleneck Block은 1×1 conv로 채널을 먼저 1/4로 줄인 뒤 3×3 conv를 수행하여 계산 효율을 유지하면서도 층을 더 깊게 쌓을 수 있게 해줍니다.

```
ResNet-34: 3.6B FLOPs  (Basic Block)
ResNet-50: 3.8B FLOPs  (Bottleneck Block, 층은 더 많지만 연산량 비슷)
```

**한 줄 요약**

> ResNet-34는 단순하지만 얕은 네트워크에 적합하고, ResNet-50은 Bottleneck으로 계산 효율을 확보하면서 더 깊고 강력한 표현력을 달성합니다.
