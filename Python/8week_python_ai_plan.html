왜 배워야 하나요?
AI가 사진을 인식할 때, 그 사진 속 숫자들은 어딘가에 담겨 있어야 해요.
변수는 숫자 하나를 담는 상자고, 리스트는 상자 수백 개를 이어붙인 서랍이에요.

코드를 묶어서 이름 붙이기 · 20문제
왜 배워야 하나요?
레시피를 생각해봐요. 매번 "밀가루 넣고, 달걀 넣고..." 반복하는 대신
"케이크 만들기"라는 이름을 붙여두면 나중에 그냥 불러서 쓰면 돼요.
함수가 바로 그 "이름 붙인 레시피"예요.
파이토치 코드는 def forward(self, x): 같은 함수들의 묶음이에요. 함수를 모르면 파이토치 코드 절반이 안 읽혀요.

lambda, map, import — AI 데이터 변환 핵심 · 15문제
왜 배워야 하나요?
AI 코드를 열면 항상 첫 줄이 import torch예요.
이게 바로 남이 만들어둔 도구 박스를 가져오는 것이에요.
또 lambda x: x * 2 같은 짧은 함수도 데이터 전처리에서 엄청 자주 나와요. 이 두 가지를 모르면 코드 첫 줄부터 막혀요.

5~6주차
클래스 · 객체지향
파이토치 모델 구조 그 자체 · 30문제
왜 배워야 하나요?
레고 설계도를 생각해봐요. 설계도 자체는 레고가 아니에요. 설계도대로 만들어야 비로소 레고가 돼요.
클래스가 설계도고, 만들어진 결과물이 객체예요.
파이토치로 AI 모델을 만들 때 class MyModel(nn.Module): 이라고 쓰는데, 이게 "내 AI 모델 설계도"를 선언하는 거예요. 클래스를 모르면 파이토치 코드는 처음부터 끝까지 암호문이에요.

300제 졸업 · 실제 AI 코드 해독 시작
드디어 여기까지 왔어요
지금까지 배운 변수, 리스트, 함수, 클래스. 이 네 가지가 파이토치 코드의 전부예요.
이제 실제 코드를 열고 "아, 이게 클래스구나. 이게 forward 함수구나" 하고 읽는 연습만 하면 돼요.
이제 이 코드가 읽혀야 해요
실제 파이토치 모델 코드
import torch import torch.nn as nn class MyModel(nn.Module): # 클래스 — 5주차 def __init__(self): # 함수 — 3주차 super().__init__() # 상속 — 5주차 self.layer = nn.Linear(10, 1) def forward(self, x): # 함수 — 3주차 return self.layer(x) # return — 3주차 model = MyModel() # 객체 생성 — 5주차 for epoch in range(100): # 반복문 — 2주차 output = model(x)
