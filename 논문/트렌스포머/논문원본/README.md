## 초록 (Abstract)

현재 주류를 이루고 있는 시퀀스 변환(sequence transduction) 모델은 인코더와 디코더를 포함하는 복잡한 순환 신경망(RNN) 또는 합성곱 신경망(CNN) 기반이다. 가장 성능이 좋은 모델들은 추가로 어텐션 메커니즘을 통해 인코더와 디코더를 연결한다. 본 논문에서는 순환과 합성곱을 완전히 배제하고 오로지 어텐션 메커니즘만으로 구성된 단순한 새 네트워크 구조인 **트랜스포머(Transformer)** 를 제안한다.

두 가지 기계 번역 과제에 대한 실험 결과, 이 모델은 품질 면에서 기존 모델을 능가하는 동시에 병렬화가 더 쉽고 학습 시간이 크게 단축됨을 보인다. 우리 모델은 WMT 2014 영어-독일어 번역 과제에서 **28.4 BLEU**를 달성해, 앙상블을 포함한 기존 최고 성능을 2 BLEU 이상 향상시켰다. WMT 2014 영어-프랑스어 번역 과제에서는 8개의 GPU에서 3.5일간 학습한 후 **41.8**이라는 단일 모델 최고 기록(BLEU)을 세웠는데, 이는 기존 최상위 모델들의 학습 비용 중 극히 일부에 불과하다. 또한 대용량 데이터와 제한된 데이터 모두에서 영어 구문 구조 분석(constituency parsing)에도 성공적으로 적용하여, 트랜스포머가 다른 과제에도 잘 일반화됨을 보인다.

## 1. 서론 (Introduction)

순환 신경망(RNN), 특히 장단기 기억(LSTM)과 게이트 순환 유닛(GRU) 신경망은 언어 모델링이나 기계 번역과 같은 시퀀스 모델링 및 변환 문제에서 최첨단 접근 방식으로 확고히 자리 잡아 왔다. 이후 순환 언어 모델과 인코더-디코더 구조의 성능을 더욱 끌어올리기 위한 수많은 연구가 이어져 왔다.

순환 모델은 일반적으로 입력과 출력 시퀀스의 기호 위치를 따라 계산을 분해한다. 위치를 계산 시간의 스텝에 대응시켜, 이전 은닉 상태 `h(t-1)`과 현재 위치 `t`의 입력을 함수로 하여 은닉 상태 `h(t)`의 시퀀스를 생성한다. 이러한 본질적으로 순차적인 성질은 학습 예제 내부에서의 병렬화를 불가능하게 만드는데, 시퀀스 길이가 길어질수록 이는 중요한 제약이 된다. 왜냐하면 메모리 제약 때문에 예제 간 배치(batch) 크기도 제한되기 때문이다. 최근에는 인수분해 기법과 조건부 계산을 통해 계산 효율성 측면에서 상당한 개선이 이루어졌고, 후자의 경우 모델 성능 향상도 함께 이루어졌다. 그러나 순차적 계산이라는 근본적 제약은 여전히 남아 있다.

어텐션 메커니즘은 다양한 과제에서 설득력 있는 시퀀스 모델링과 변환 모델의 필수 구성 요소가 되었으며, 입력 또는 출력 시퀀스 내에서 거리에 관계없이 의존성을 모델링할 수 있게 해준다. 그러나 극히 일부의 경우를 제외하면, 이러한 어텐션 메커니즘은 순환 신경망과 결합되어 사용된다.

본 논문에서는 순환을 배제하고 입력과 출력 사이의 전역적 의존성을 어텐션 메커니즘만으로 끌어내는 모델 구조인 **트랜스포머**를 제안한다. 트랜스포머는 훨씬 더 높은 수준의 병렬화를 가능하게 하며, 8개의 P100 GPU에서 불과 12시간 학습만으로 번역 품질에서 새로운 최첨단 수준에 도달할 수 있다.

## 2. 배경 (Background)

순차적 계산을 줄이려는 목표는 Extended Neural GPU, ByteNet, ConvS2S 등의 기반이기도 하다. 이들은 모두 합성곱 신경망을 기본 구성 요소로 사용하여, 모든 입력 및 출력 위치에 대한 은닉 표현을 병렬로 계산한다. 이러한 모델에서 임의의 두 입력 또는 출력 위치 사이의 신호를 연결하는 데 필요한 연산 수는 위치 간 거리에 따라 증가하는데, ConvS2S는 선형으로, ByteNet은 로그 형태로 증가한다. 이는 멀리 떨어진 위치 간의 의존성을 학습하기 어렵게 만든다. 트랜스포머에서는 이 연산 수가 **상수 개수**로 축소된다. 다만 어텐션 가중 위치의 평균화로 인해 유효 해상도가 감소한다는 대가를 치르는데, 우리는 이를 3.2절에서 설명하는 멀티헤드 어텐션(Multi-Head Attention)으로 보완한다.

**셀프 어텐션(self-attention)** 은 '인트라 어텐션'이라고도 불리며, 단일 시퀀스의 서로 다른 위치들을 서로 연결하여 해당 시퀀스의 표현을 계산하는 어텐션 메커니즘이다. 셀프 어텐션은 독해, 요약, 텍스트 함의, 과제 독립적 문장 표현 학습 등 다양한 과제에서 성공적으로 활용되어 왔다.

종단간(end-to-end) 메모리 네트워크는 시퀀스 정렬 순환 대신 순환 어텐션 메커니즘을 기반으로 하며, 단순 언어 질의응답 및 언어 모델링 과제에서 좋은 성능을 보이는 것으로 알려져 있다.

우리가 아는 한, 트랜스포머는 시퀀스 정렬 RNN이나 합성곱을 사용하지 않고 **오로지 셀프 어텐션만으로** 입력과 출력의 표현을 계산하는 최초의 변환 모델이다.

## 3. 모델 구조 (Model Architecture)

가장 경쟁력 있는 신경망 시퀀스 변환 모델들은 대부분 인코더-디코더 구조를 가진다. 여기서 인코더는 기호 표현의 입력 시퀀스 `(x₁, ..., xₙ)`을 연속 표현 시퀀스 `z = (z₁, ..., zₙ)`으로 매핑한다. 주어진 `z`로부터 디코더는 출력 시퀀스 `(y₁, ..., yₘ)`을 한 번에 하나의 기호씩 생성한다. 각 단계에서 모델은 자기회귀적(auto-regressive)으로 동작하여, 다음 기호를 생성할 때 이전에 생성된 기호들을 추가 입력으로 사용한다.

트랜스포머는 인코더와 디코더 모두에 대해 스택 형태의 셀프 어텐션과 위치별 완전 연결층(point-wise, fully connected layers)을 사용하는 전체 구조를 따른다.

### 3.1 인코더 및 디코더 스택 (Encoder and Decoder Stacks)

**인코더 (Encoder)**

인코더는 `N = 6`개의 동일한 층을 쌓아 구성된다. 각 층은 두 개의 하위 층(sub-layer)으로 이루어져 있다. 첫 번째는 멀티헤드 셀프 어텐션 메커니즘이고, 두 번째는 단순한 위치별 완전 연결 피드포워드 네트워크이다. 우리는 두 하위 층 각각에 **잔차 연결(residual connection)** 을 적용하고 그 뒤에 **층 정규화(layer normalization)** 를 둔다. 즉, 각 하위 층의 출력은 다음과 같다:

```
LayerNorm(x + Sublayer(x))
```

여기서 `Sublayer(x)`는 해당 하위 층이 수행하는 함수이다. 이러한 잔차 연결을 가능하게 하기 위해 모델의 모든 하위 층과 임베딩 층은 차원 `d_model = 512`의 출력을 생성한다.

**디코더 (Decoder)**

디코더도 `N = 6`개의 동일한 층을 쌓아 구성된다. 인코더 층의 두 하위 층에 더해, 디코더는 인코더 스택의 출력에 대해 멀티헤드 어텐션을 수행하는 **세 번째 하위 층**을 삽입한다. 인코더와 마찬가지로 각 하위 층에 잔차 연결과 층 정규화를 적용한다. 또한 디코더 스택의 셀프 어텐션 하위 층은 현재 위치가 이후 위치를 참조하지 못하도록 **마스킹(masking)** 된다. 이 마스킹과 출력 임베딩을 한 위치씩 오른쪽으로 밀어두는 것을 결합함으로써, 위치 `i`에 대한 예측이 `i`보다 이전 위치의 알려진 출력에만 의존하도록 보장한다.

### 3.2 어텐션 (Attention)

어텐션 함수는 하나의 **쿼리(query)** 와 **키-값(key-value) 쌍**들의 집합을 출력으로 매핑하는 함수라고 설명할 수 있다. 여기서 쿼리, 키, 값, 출력은 모두 벡터이다. 출력은 값들의 **가중합**으로 계산되며, 각 값에 할당되는 가중치는 해당 쿼리와 대응되는 키 사이의 호환성(compatibility) 함수로 계산된다.

#### 3.2.1 스케일드 닷-프로덕트 어텐션 (Scaled Dot-Product Attention)

우리가 사용하는 어텐션을 '스케일드 닷-프로덕트 어텐션'이라고 부른다. 입력은 차원 `d_k`인 쿼리와 키, 그리고 차원 `d_v`인 값들로 구성된다. 쿼리와 모든 키의 내적(dot product)을 계산한 뒤, 각각을 `√d_k`로 나누고, 소프트맥스(softmax) 함수를 적용하여 값에 대한 가중치를 얻는다.

실제 구현에서는 여러 쿼리에 대해 어텐션 함수를 동시에 계산하기 위해 쿼리들을 하나의 행렬 `Q`로 묶는다. 키와 값도 각각 행렬 `K`와 `V`로 묶는다. 출력 행렬은 다음과 같이 계산된다:

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V \tag{1}
$$

가장 흔하게 사용되는 두 가지 어텐션 함수는 **가산적(additive) 어텐션**과 **내적(곱셈적) 어텐션**이다. 내적 어텐션은 스케일링 계수 `1/√d_k`가 없다는 점만 제외하면 우리 알고리즘과 동일하다. 가산적 어텐션은 은닉층이 하나인 피드포워드 네트워크로 호환성 함수를 계산한다. 두 방식은 이론적 복잡도 면에서 유사하지만, 실제로는 내적 어텐션이 고도로 최적화된 행렬 곱셈 코드로 구현될 수 있기 때문에 훨씬 빠르고 공간 효율적이다.

`d_k` 값이 작은 경우에는 두 메커니즘의 성능이 비슷하지만, `d_k` 값이 클 때 스케일링을 하지 않으면 가산적 어텐션이 내적 어텐션보다 성능이 우수하다. 우리는 `d_k`가 클 때 내적 값의 크기가 커져서 소프트맥스 함수가 매우 작은 기울기를 갖는 영역으로 밀려나는 현상이 원인이라고 추측한다. 이 효과를 상쇄하기 위해 내적 값을 `1/√d_k`로 스케일링한다.

#### 3.2.2 멀티헤드 어텐션 (Multi-Head Attention)

`d_model` 차원의 키, 값, 쿼리에 대해 단 한 번의 어텐션 함수를 수행하는 대신, 쿼리, 키, 값을 각각 학습된 서로 다른 선형 투영(linear projection)을 통해 `d_k`, `d_k`, `d_v` 차원으로 `h`번 투영하는 것이 유익함을 발견했다. 투영된 각 쿼리, 키, 값 버전에 대해 어텐션 함수를 병렬로 수행하여 `d_v` 차원의 출력 값을 얻는다. 이들을 이어붙인(concat) 뒤 다시 한 번 투영하여 최종 값을 만든다.

**멀티헤드 어텐션은 모델이 서로 다른 위치에 있는 서로 다른 표현 부분공간(representation subspaces)의 정보에 동시에 주의를 기울일 수 있게 해준다.** 단일 어텐션 헤드 하나만 있으면 평균화 효과 때문에 이러한 효과가 억제된다.

$$
\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, ..., \text{head}_h) W^O
$$

$$
\text{where} \quad \text{head}_i = \text{Attention}(Q W_i^Q, K W_i^K, V W_i^V)
$$

투영 행렬은 다음과 같다:
- $W_i^Q \in \mathbb{R}^{d_{\text{model}} \times d_k}$
- $W_i^K \in \mathbb{R}^{d_{\text{model}} \times d_k}$
- $W_i^V \in \mathbb{R}^{d_{\text{model}} \times d_v}$
- $W^O \in \mathbb{R}^{h \cdot d_v \times d_{\text{model}}}$

본 연구에서는 `h = 8`개의 병렬 어텐션 층(헤드)을 사용한다. 각각에 대해 `d_k = d_v = d_model / h = 64`를 사용한다. 각 헤드의 차원이 줄어들기 때문에, 전체 계산 비용은 전체 차원을 가진 단일 헤드 어텐션과 비슷하다.

#### 3.2.3 모델에서의 어텐션 활용 (Applications of Attention in our Model)

트랜스포머는 세 가지 방식으로 멀티헤드 어텐션을 사용한다:

- **인코더-디코더 어텐션 층**: 쿼리는 이전 디코더 층에서 오고, 메모리 키와 값은 인코더의 출력에서 온다. 이를 통해 디코더의 모든 위치는 입력 시퀀스의 모든 위치에 주의를 기울일 수 있다. 이는 기존 시퀀스-투-시퀀스 모델의 전형적인 인코더-디코더 어텐션을 모방한 것이다.

- **인코더의 셀프 어텐션 층**: 키, 값, 쿼리가 모두 같은 곳, 즉 인코더의 이전 층의 출력에서 온다. 인코더의 각 위치는 이전 층의 모든 위치에 주의를 기울일 수 있다.

- **디코더의 셀프 어텐션 층**: 디코더의 각 위치가 자기 자신을 포함한 이전 위치까지 모든 위치에 주의를 기울이도록 한다. 디코더에서 왼쪽 방향(미래)으로의 정보 흐름을 막아 자기회귀 성질을 보존해야 한다. 우리는 이를 스케일드 닷-프로덕트 어텐션 내부에서 소프트맥스 입력 중 허용되지 않는 연결에 해당하는 모든 값을 `-∞`로 마스킹하는 방식으로 구현한다.

### 3.3 위치별 피드포워드 네트워크 (Position-wise Feed-Forward Networks)

어텐션 하위 층 외에도 인코더와 디코더의 각 층은 완전 연결 피드포워드 네트워크를 포함하며, 이는 각 위치에 대해 개별적으로 동일하게 적용된다. 이 네트워크는 두 개의 선형 변환과 그 사이에 적용되는 ReLU 활성화로 구성된다.

$$
\text{FFN}(x) = \max(0, x W_1 + b_1) W_2 + b_2 \tag{2}
$$

선형 변환은 서로 다른 위치에서 동일하게 적용되지만, 층마다 다른 파라미터를 사용한다. 이는 커널 크기가 1인 두 번의 합성곱으로도 표현할 수 있다. 입력과 출력의 차원은 `d_model = 512`이고, 내부 층의 차원은 `d_ff = 2048`이다.

### 3.4 임베딩과 소프트맥스 (Embeddings and Softmax)

다른 시퀀스 변환 모델들과 마찬가지로, 우리는 학습된 임베딩을 사용해 입력 토큰과 출력 토큰을 `d_model` 차원의 벡터로 변환한다. 또한 보통의 학습된 선형 변환과 소프트맥스 함수를 사용해 디코더 출력을 다음 토큰의 예측 확률로 변환한다. 우리 모델에서는 두 임베딩 층과 소프트맥스 이전의 선형 변환이 **동일한 가중치 행렬을 공유**한다. 임베딩 층에서는 그 가중치에 `√d_model`을 곱한다.

### 3.5 위치 인코딩 (Positional Encoding)

우리 모델은 순환도 합성곱도 포함하지 않기 때문에, 시퀀스의 순서를 활용하려면 토큰들의 상대적 또는 절대적 위치 정보를 주입해야 한다. 이를 위해 인코더와 디코더 스택 하단에서 입력 임베딩에 **위치 인코딩(positional encoding)** 을 더한다. 위치 인코딩은 임베딩과 동일한 차원 `d_model`을 가지므로 둘을 더할 수 있다.

본 연구에서는 서로 다른 주파수의 사인·코사인 함수를 사용한다:

$$
PE_{(pos, 2i)} = \sin(pos / 10000^{2i / d_{\text{model}}})
$$

$$
PE_{(pos, 2i+1)} = \cos(pos / 10000^{2i / d_{\text{model}}})
$$

여기서 `pos`는 위치이고 `i`는 차원이다. 즉 위치 인코딩의 각 차원은 하나의 사인파에 해당한다. 파장은 `2π`에서 `10000 · 2π`까지의 등비수열을 이룬다. 우리가 이 함수를 선택한 이유는, 임의의 고정된 오프셋 `k`에 대해 `PE(pos+k)`가 `PE(pos)`의 선형 함수로 표현될 수 있어 모델이 상대적 위치로 어텐션을 학습하기 쉬울 것이라 가정했기 때문이다.

학습되는 위치 임베딩도 실험해 보았는데, 두 방식이 거의 동일한 결과를 낸다는 것을 확인했다. 사인파 방식을 택한 이유는, 학습 시 보았던 시퀀스보다 더 긴 시퀀스 길이로의 외삽(extrapolation)이 가능할 수 있기 때문이다.

## 4. 왜 셀프 어텐션인가 (Why Self-Attention)

이 절에서는 셀프 어텐션 층을 순환 층·합성곱 층과 비교한다. 셀프 어텐션 사용의 동기로 세 가지 기준을 고려한다:

1. **층당 전체 계산 복잡도**
2. **병렬화 가능한 계산량** (요구되는 최소 순차 연산 수로 측정)
3. **네트워크 내 장거리 의존성 간의 경로 길이**

장거리 의존성 학습은 많은 시퀀스 변환 과제에서 핵심적인 도전 과제이다. 순전파와 역전파 신호가 네트워크 내에서 지나가야 하는 경로가 짧을수록 장거리 의존성 학습이 쉬워진다.

### 📊 표 1. 층 유형별 복잡도 비교

> `n`: 시퀀스 길이, `d`: 표현 차원, `k`: 합성곱 커널 크기, `r`: 제한된 셀프 어텐션의 이웃 크기

| 층 유형 (Layer Type) | 층당 복잡도 | 순차 연산 수 | 최대 경로 길이 |
|---|---|---|---|
| **셀프 어텐션** | O(n² · d) | **O(1)** | **O(1)** |
| 순환 (Recurrent) | O(n · d²) | O(n) | O(n) |
| 합성곱 (Convolutional) | O(k · n · d²) | O(1) | O(logₖ(n)) |
| 셀프 어텐션 (제한됨) | O(r · n · d) | O(1) | O(n/r) |

셀프 어텐션 층은 모든 위치를 **상수 개의 순차 연산**으로 연결하는 반면, 순환 층은 `O(n)`의 순차 연산이 필요하다. 계산 복잡도 면에서 셀프 어텐션 층은, 시퀀스 길이 `n`이 표현 차원 `d`보다 작을 때 순환 층보다 빠른데, 이는 최첨단 기계 번역 모델이 사용하는 워드피스(word-piece)나 바이트 페어(byte-pair) 표현에서 대부분 해당되는 경우이다.

커널 너비 `k < n`인 단일 합성곱 층은 모든 입력·출력 위치 쌍을 연결하지 못한다. 연속 커널의 경우 `O(n/k)` 개의 합성곱 층 스택이 필요하고, 확장(dilated) 합성곱의 경우 `O(logₖ(n))` 개의 층이 필요하여, 네트워크 내 임의의 두 위치 사이의 최장 경로가 늘어난다. 합성곱 층은 일반적으로 순환 층보다 `k`배 정도 더 비싸다.

부수적 이점으로, 셀프 어텐션은 **더 해석 가능한 모델**을 만들 수 있다. 개별 어텐션 헤드들이 서로 다른 작업을 수행하도록 명확히 학습될 뿐 아니라, 문장의 구문적·의미적 구조와 관련된 것처럼 보이는 행동을 나타내는 헤드들이 많다.

## 5. 학습 (Training)

### 5.1 학습 데이터 및 배칭 (Training Data and Batching)

- **WMT 2014 영어-독일어**: 약 450만 개의 문장 쌍, BPE로 인코딩(약 37,000 토큰 공유 어휘)
- **WMT 2014 영어-프랑스어**: 3,600만 문장, 워드피스 32,000개 어휘
- 각 학습 배치: 소스/타깃 각각 약 25,000 토큰

### 5.2 하드웨어 및 일정 (Hardware and Schedule)

- **하드웨어**: NVIDIA P100 GPU × 8
- **베이스 모델**: 스텝당 약 0.4초, 총 100,000 스텝 (12시간)
- **빅 모델**: 스텝당 1.0초, 총 300,000 스텝 (3.5일)

### 5.3 옵티마이저 (Optimizer)

**Adam 옵티마이저** (`β₁ = 0.9`, `β₂ = 0.98`, `ε = 10⁻⁹`)를 사용하며, 학습률은 다음 공식에 따라 변화시킨다:

$$
lrate = d_{\text{model}}^{-0.5} \cdot \min(\text{step\_num}^{-0.5}, \text{step\_num} \cdot \text{warmup\_steps}^{-1.5}) \tag{3}
$$

처음 `warmup_steps` 동안은 학습률을 선형으로 증가시키고, 그 이후에는 스텝 번호의 역제곱근에 비례해 감소시킨다. `warmup_steps = 4000`을 사용했다.

### 5.4 정규화 (Regularization)

학습 중 세 가지 정규화 기법을 적용한다:

- **잔차 드롭아웃 (Residual Dropout)**: 각 하위 층의 출력이 하위 층 입력에 더해져 정규화되기 전에 드롭아웃 적용. 임베딩과 위치 인코딩의 합에도 적용. 베이스 모델 `P_drop = 0.1`.
- **레이블 스무딩 (Label Smoothing)**: `ε_ls = 0.1`. 퍼플렉시티는 나빠지지만 정확도와 BLEU는 향상된다.

## 6. 결과 (Results)

### 6.1 기계 번역 (Machine Translation)

WMT 2014 영어-독일어 번역 과제에서 빅 트랜스포머 모델은 앙상블을 포함한 기존 최고 모델들을 **2.0 BLEU 이상 능가**하며, 28.4라는 새로운 최첨단 BLEU를 기록했다. 학습은 8개의 P100 GPU에서 3.5일이 걸렸다. WMT 2014 영어-프랑스어에서는 41.0 BLEU로 기존 최고 모델 대비 **1/4 미만의 학습 비용**으로 더 좋은 성능을 달성했다.

#### 📊 표 2. BLEU 점수 및 학습 비용 비교

| 모델 | BLEU (EN-DE) | BLEU (EN-FR) | 학습 FLOPs (EN-DE) | 학습 FLOPs (EN-FR) |
|---|---|---|---|---|
| ByteNet | 23.75 | — | — | — |
| GNMT + RL | 24.6 | 39.92 | 2.3·10¹⁹ | 1.4·10²⁰ |
| ConvS2S | 25.16 | 40.46 | 9.6·10¹⁸ | 1.5·10²⁰ |
| MoE | 26.03 | 40.56 | 2.0·10¹⁹ | 1.2·10²⁰ |
| GNMT + RL 앙상블 | 26.30 | 41.16 | 1.8·10²⁰ | 1.1·10²¹ |
| ConvS2S 앙상블 | 26.36 | 41.29 | 7.7·10¹⁹ | 1.2·10²¹ |
| **Transformer (베이스)** | **27.3** | **38.1** | **3.3·10¹⁸** | **3.3·10¹⁸** |
| **Transformer (빅)** | **28.4** | **41.8** | **2.3·10¹⁹** | **2.3·10¹⁹** |

추론 시에는 빔 크기 4, 길이 패널티 `α = 0.6`의 빔 서치를 사용했다. 베이스 모델은 마지막 5개 체크포인트 평균, 빅 모델은 마지막 20개 체크포인트 평균을 사용했다.

### 6.2 모델 변형 (Model Variations)

트랜스포머의 여러 구성 요소의 중요성을 평가하기 위해 베이스 모델을 여러 방식으로 변형하여 성능 변화를 측정했다.

- **헤드 수 (A)**: 단일 헤드 어텐션은 최적 설정보다 0.9 BLEU 낮다. 헤드 수가 너무 많아도 품질이 떨어진다.
- **키 크기 (B)**: `d_k`를 줄이면 모델 품질이 떨어진다. 내적보다 더 정교한 호환성 함수가 도움이 될 수 있음을 시사.
- **모델 크기 (C) / 드롭아웃 (D)**: 예상대로 큰 모델이 더 낫고, 드롭아웃은 과적합 방지에 매우 유용하다.
- **위치 인코딩 (E)**: 사인파 방식을 학습되는 위치 임베딩으로 교체해도 거의 동일한 결과.

### 6.3 영어 구문 구조 분석 (English Constituency Parsing)

트랜스포머의 일반화 능력을 평가하기 위해 영어 구문 구조 분석에 적용했다. Penn Treebank의 WSJ 부분(약 4만 문장)에 `d_model = 1024`의 4층 트랜스포머를 학습했다. 과제별 튜닝이 거의 없었음에도 불구하고 놀라울 정도로 좋은 성능을 보였으며, Recurrent Neural Network Grammar를 제외한 모든 이전 모델보다 뛰어난 결과를 얻었다. 특히 WSJ 학습 세트 4만 문장만으로 학습할 때에도 BerkeleyParser를 능가했다.

## 7. 결론 (Conclusion)

본 연구에서 우리는 **오로지 어텐션만을 기반으로 한 최초의 시퀀스 변환 모델**인 트랜스포머를 제시했다. 이는 인코더-디코더 구조에서 가장 흔히 사용되던 순환 층을 멀티헤드 셀프 어텐션으로 대체한다.

번역 과제에서 트랜스포머는 순환 또는 합성곱 층에 기반한 구조보다 훨씬 빠르게 학습할 수 있다. WMT 2014 영어-독일어와 영어-프랑스어 번역 과제 모두에서 새로운 최첨단 성능을 달성했다.

우리는 어텐션 기반 모델의 미래에 대해 기대가 크며, 이를 다른 과제들에도 적용할 계획이다. 텍스트 이외의 입력·출력 양식(modality)을 포함하는 문제로 트랜스포머를 확장하고, **이미지·오디오·비디오** 같은 큰 입력·출력을 효율적으로 처리하기 위해 국소적이고 제한된 어텐션 메커니즘을 조사할 계획이다. 생성의 순차성을 줄이는 것도 우리의 또 다른 연구 목표이다.

### 핵심 용어 한눈에 보기

| 영어 | 번역 | 쉬운 설명 |
|---|---|---|
| Attention | 어텐션 / 주의 | 입력의 어느 부분에 얼마나 '집중'할지 학습하는 메커니즘 |
| Self-Attention | 셀프 어텐션 | 문장 안에서 단어끼리 서로를 참조하는 어텐션 |
| Multi-Head | 멀티헤드 | 여러 시각에서 동시에 관계를 보는 병렬 어텐션 |
| Query/Key/Value | 쿼리/키/값 | '찾고자 하는 것 / 검색 키 / 실제 내용'으로 이해하면 됨 |
| Encoder/Decoder | 인코더/디코더 | '입력 이해기 / 출력 생성기'의 2단 구조 |
| Positional Encoding | 위치 인코딩 | 단어 순서 정보를 sin/cos 함수로 주입 |
| Residual Connection | 잔차 연결 | 입력을 출력에 그대로 더해 학습 안정화 (ResNet과 동일 개념) |
| Layer Normalization | 층 정규화 | 각 층의 출력을 정규화해 학습 안정화 |
| BLEU | BLEU 점수 | 기계 번역 품질 평가 지표 (높을수록 좋음) |

### 영상 처리 프로젝트와의 연결 고리

이 논문은 기계 번역을 다루지만, 여기서 제안된 트랜스포머 구조는 이후 영상 분야로 그대로 확장됩니다.

- **VideoMAE / TimeSformer**: 영상의 프레임을 일련의 '시각적 단어'처럼 취급하고, 셀프 어텐션을 시공간(spatio-temporal) 패치에 적용합니다.
- **TimeSformer**는 시간 축 어텐션과 공간 축 어텐션을 분리해 적용하는데, 이는 본 논문의 **멀티헤드 어텐션 아이디어**를 시공간으로 확장한 것입니다.
- **위치 인코딩**(3.5절)은 영상에서 프레임 순서와 패치 위치를 표현하는 데 똑같이 사용됩니다.

> 💡 3.2절(스케일드 닷-프로덕트 어텐션)과 3.5절(위치 인코딩)만 제대로 이해해도, 영상 Transformer 모델들이 왜 그렇게 설계되었는지 자연스럽게 보입니다.

### 추천 읽기 순서

처음 읽는다면 아래 순서를 추천합니다:

1. **3.2.1절 스케일드 닷-프로덕트 어텐션** — 트랜스포머의 심장부. 수식 (1)만 이해해도 논문의 절반을 이해한 것.
2. **3.2.2절 멀티헤드 어텐션** — 왜 `h=8`개의 헤드로 나누는지. *'여러 관점에서 동시에 보기'* 라는 직관이 중요.
3. **4절 왜 셀프 어텐션인가** — RNN/CNN과 비교해 왜 더 빠르고 병렬화가 잘 되는지. **표 1**이 핵심.
4. **5.3절 옵티마이저** — `warmup` + 역제곱근 감쇠 스케줄은 지금도 트랜스포머 학습에서 널리 쓰임.
