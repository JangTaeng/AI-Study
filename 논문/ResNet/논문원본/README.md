# 이미지 인식을 위한 딥 잔차 학습 (Deep Residual Learning for Image Recognition)

> **저자:** Kaiming He, Xiangyu Zhang, Shaoqing Ren, Jian Sun  
> **소속:** Microsoft Research  
> **원본 논문:** arXiv:1512.03385v1 [cs.CV] (2015년 12월 10일)

---

## 초록

더 깊은 신경망은 학습시키기가 더 어렵습니다. 본 논문에서는 기존에 사용되던 것보다 훨씬 깊은 신경망의 학습을 용이하게 하기 위한 **잔차 학습(Residual Learning) 프레임워크**를 제안합니다.

우리는 레이어들이 참조되지 않은 함수를 직접 학습하는 대신, **레이어 입력을 기준으로 한 잔차 함수**를 학습하도록 명시적으로 재구성합니다. 이러한 잔차 네트워크가 최적화하기 더 쉽고, 깊이가 상당히 증가해도 정확도를 향상시킬 수 있음을 보여주는 포괄적인 실험 결과를 제공합니다.

ImageNet 데이터셋에서 최대 **152개 레이어** 깊이의 잔차 네트워크를 평가했으며, 이는 VGG 네트워크보다 8배 깊지만 복잡도는 오히려 낮습니다. 이 잔차 네트워크들의 앙상블은 ImageNet 테스트 셋에서 **3.57% 오류율**을 달성하여 ILSVRC 2015 분류 과제에서 **1위**를 차지했습니다.

---

## 1. 서론

깊은 합성곱 신경망(CNN)은 이미지 분류 분야에서 혁신적인 성과를 이끌어왔습니다. 네트워크의 깊이가 중요하다는 사실은 이미 잘 알려져 있으나, 단순히 레이어를 쌓는 것만으로는 더 좋은 네트워크를 학습시킬 수 없습니다.

### 핵심 문제: 성능 저하(Degradation) 문제

네트워크 깊이가 증가할수록 다음과 같은 문제가 발생합니다:

- 정확도가 포화 상태에 이른 후 급격히 **저하**됩니다.
- 이 저하는 **과적합(overfitting)으로 인한 것이 아닙니다.**
- 깊은 모델에 레이어를 더 추가하면 오히려 **학습 오류가 증가**합니다.

> 예시: CIFAR-10에서 56층 네트워크가 20층 네트워크보다 학습 오류와 테스트 오류 모두 높게 나타납니다.

## 2. 잔차 학습 (Deep Residual Learning)

### 2.1 핵심 아이디어

원하는 기본 매핑을 `H(x)`라고 할 때, 쌓인 비선형 레이어들이 다음과 같은 **잔차 함수**를 학습하도록 명시적으로 설정합니다:

```
F(x) := H(x) - x
```

원래의 함수는 `F(x) + x`로 재구성됩니다. **항등 매핑(identity mapping)이 최적일 경우**, 잔차를 0으로 만드는 것이 비선형 레이어들로 항등 함수를 직접 학습하는 것보다 훨씬 쉽습니다.

### 2.2 잔차 블록 구조

```
입력 x
  │
  ├──────────────────┐  (shortcut connection)
  │                  │
  ▼                  │
[가중치 레이어]       │
  │                  │
  ▼                  │
[ReLU]               │
  │                  │
  ▼                  │
[가중치 레이어]       │
  │                  │
  └──────────────────┤
                     ▼
               F(x) + x  (element-wise addition)
                     │
                     ▼
                  [ReLU]
                출력 y
```

### 수식

$$y = F(x, \{W_i\}) + x$$

- 차원이 다를 경우: $y = F(x, \{W_i\}) + W_s x$

### 2.3 숏컷 연결(Shortcut Connections)의 장점

- **추가 파라미터 없음**: 항등 숏컷은 연산 복잡도를 증가시키지 않습니다.
- **역전파 용이**: 전체 네트워크를 SGD + 역전파로 end-to-end 학습 가능합니다.
- **구현 간단**: Caffe 등 일반 라이브러리로 쉽게 구현 가능합니다.

## 3. 네트워크 구조

### 3.1 Plain 네트워크 (기준선)

VGG 네트워크 철학에서 영감을 받았으며, 두 가지 설계 규칙을 따릅니다:
1. 출력 특성 맵 크기가 같으면 → 필터 수 동일
2. 특성 맵 크기가 절반이 되면 → 필터 수를 두 배로

### 3.2 잔차 네트워크 (ResNet)

Plain 네트워크에 숏컷 연결을 추가한 버전입니다.

| 레이어명 | 출력 크기 | 18-layer | 34-layer | 50-layer | 101-layer | 152-layer |
|---------|---------|---------|---------|---------|----------|----------|
| conv1   | 112×112 | 7×7, 64, stride 2 | | | | |
| conv2_x | 56×56  | [3×3, 64] ×2 | [3×3, 64] ×3 | [1×1,64 / 3×3,64 / 1×1,256] ×3 | ×3 | ×3 |
| conv3_x | 28×28  | [3×3, 128] ×2 | [3×3, 128] ×4 | [1×1,128 / 3×3,128 / 1×1,512] ×4 | ×4 | ×8 |
| conv4_x | 14×14  | [3×3, 256] ×2 | [3×3, 256] ×6 | [1×1,256 / 3×3,256 / 1×1,1024] ×6 | ×23 | ×36 |
| conv5_x | 7×7    | [3×3, 512] ×2 | [3×3, 512] ×3 | [1×1,512 / 3×3,512 / 1×1,2048] ×3 | ×3 | ×3 |
| -       | 1×1    | average pool, 1000-d fc, softmax | | | | |
| **FLOPs** | - | 1.8×10⁹ | 3.6×10⁹ | 3.8×10⁹ | 7.6×10⁹ | 11.3×10⁹ |

### 3.3 보틀넥(Bottleneck) 구조

50층 이상의 깊은 네트워크에서는 학습 시간 효율을 위해 **보틀넥 블록**을 사용합니다:

```
[1×1 conv]  → 차원 축소
[3×3 conv]  → 주요 연산
[1×1 conv]  → 차원 복원
```

## 5. 실험 결과

### 5.1 ImageNet 분류

#### Plain 네트워크 vs ResNet (Top-1 오류율 비교)

| 모델 | Plain | ResNet |
|-----|-------|--------|
| 18층 | 27.94% | 27.88% |
| 34층 | 28.54% | **25.03%** |

- Plain 네트워크: 34층이 18층보다 **오류율이 높음** (성능 저하 문제)
- ResNet: 34층이 18층보다 **2.8% 더 좋음** (깊어질수록 성능 향상)

#### 다양한 ResNet 깊이 비교

| 모델 | Top-1 오류율 | Top-5 오류율 |
|-----|------------|------------|
| VGG-16 | 28.07% | 9.33% |
| ResNet-34 B | 24.52% | 7.46% |
| ResNet-50 | 22.85% | 6.71% |
| ResNet-101 | 21.75% | 6.05% |
| ResNet-152 | **21.43%** | **5.71%** |

#### 앙상블 결과 (ILSVRC 2015)

| 방법 | Top-5 오류율 (테스트) |
|-----|-------------------|
| GoogLeNet (ILSVRC'14) | 6.66% |
| **ResNet 앙상블 (ILSVRC'15)** | **3.57%** 🏆 |

### 5.2 CIFAR-10 분류

| 모델 | 레이어 수 | 파라미터 수 | 오류율 |
|-----|---------|-----------|------|
| Highway Network | 19 | 2.3M | 7.54% |
| ResNet | 20 | 0.27M | 8.75% |
| ResNet | 56 | 0.85M | 6.97% |
| ResNet | **110** | 1.7M | **6.43%** |
| ResNet | 1202 | 19.4M | 7.93%* |

> *1202층 모델은 소규모 데이터셋에 대한 과적합으로 인해 110층보다 성능이 낮음

### 5.3 객체 검출 (Object Detection)

Faster R-CNN 기반으로 VGG-16을 ResNet-101로 교체했을 때:

#### PASCAL VOC

| 백본 모델 | mAP (VOC 2007) | mAP (VOC 2012) |
|--------|--------------|--------------|
| VGG-16 | 73.2% | 70.4% |
| ResNet-101 | **76.4%** | **73.8%** |

#### MS COCO

| 백본 모델 | mAP@.5 | mAP@[.5,.95] |
|--------|--------|------------|
| VGG-16 | 41.5% | 21.2% |
| ResNet-101 | **48.4%** | **27.2%** |

> ResNet-101은 COCO 표준 지표에서 **28% 상대적 향상** 달성

## 6. 주요 발견 및 분석

### 레이어 응답 분석

- ResNet은 Plain 네트워크에 비해 레이어 응답의 표준편차가 **전반적으로 더 작음**
- 더 깊은 ResNet일수록 개별 레이어의 신호 수정 정도가 **더 작음**
- 이는 잔차 함수가 0에 가깝게 학습된다는 가설을 뒷받침함

### 핵심 결론

1. **성능 저하 문제 해결**: ResNet은 깊이가 깊어져도 성능이 저하되지 않음
2. **깊이 = 성능 향상**: 잔차 학습 덕분에 깊을수록 더 좋은 결과
3. **범용성**: 분류, 검출, 위치 추정 등 다양한 비전 과제에서 효과적
4. **효율성**: 152층 ResNet도 VGG보다 낮은 연산 복잡도 유지
